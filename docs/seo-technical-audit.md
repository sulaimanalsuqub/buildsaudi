# Technical SEO Audit

Stack confirmed by inspection: Next.js 15.5 (App Router), React 19, TypeScript (strict), Tailwind CSS, deployed on Vercel. Package manager: npm (`package-lock.json`). No CMS; content is hardcoded in TSX. No test suite exists (`package.json` has no `test` script). ESLint via `eslint-config-next`. `serverExternalPackages: ["pdf-parse", "xlsx"]` in `next.config.ts` for the vendor-onboarding document parsing (unrelated to public site rendering).

Routing: `app/(site)/*` = English at path root (`/`), `app/ar/*` = Arabic at `/ar` prefix. No `/en` prefix exists (confirmed 404) ‚Äî this is the intended design, not a bug. Middleware (`middleware.ts`) redirects `/admin` ‚Üí `ODOO_BASE_URL`, and gates the whole site behind `/maintenance` when `NEXT_PUBLIC_MAINTENANCE_MODE=true` (currently `false` in `.env.local` ‚Äî site is live, not in maintenance).
