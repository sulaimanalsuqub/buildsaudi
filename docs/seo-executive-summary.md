# الملخص التنفيذي — تدقيق السيو الفني والذكاء الاصطناعي لموقع Build

**الفرع:** `seo/technical-and-ai-search-optimization` — لم يُدمج بـ `main` ولم يُنشر للإنتاج.

## لماذا عدد الصفحات المفهرسة قليل (4 من ~14)؟

**السبب الجذري المؤكد لم يُحدَّد بشكل قاطع** لأن التدقيق تم بدون وصول مباشر لـ Google Search Console — الأرقام (4 مفهرسة/11 غير مفهرسة) معلومة من المالك فقط. ما تم تأكيده وما بقي افتراضًا موضّح أدناه.

### مشاكل فنية مؤكدة (وليست بالضرورة سبب ضعف الفهرسة)

1. **كل صفحات الموقع كانت "ديناميكية" إجباريًا بلا أي كاش** — بسبب استدعاء `headers()` في الـroot layout لتحديد اللغة فقط. مؤكَّد عبر `next build` (كل الصفحات ƒ Dynamic) وheaders الاستجابة الحية (`Cache-Control: private, no-store`). **هذا تصنيفه: مشكلة تقنية/أداء مؤكدة، وقد يؤثر على كفاءة الزحف، لكنه ليس سببًا مؤكدًا لاستبعاد الفهرسة** إلا إذا ربطناه ببيانات URL Inspection أو Crawl Stats من Search Console — **تم إصلاحه بالكامل بهذا الفرع.**
2. **`dir="rtl"` مفقود من وسم `<html>`** بصفحات العربي (كان موجود بس على عنصر داخلي). **تم إصلاحه.**
3. **إعادة توجيه `https://build.sa` (بدون www) مؤقتة (307) بدل دائمة** — إعداد على مستوى Vercel، خارج الريبو، لم يُصلح بهذا التمرير.
4. لا توجد قواعد صريحة لـOAI-SearchBot أو GPTBot بملف robots.txt — لكن كلاهما **غير محظور فعليًا حاليًا** (مسموح ضمنيًا عبر القاعدة العامة).

### مشاكل مستبعدة تمامًا (مؤكد، مو مجرد شك)

الموقع غير محظور بـrobots.txt، لا يوجد noindex، الـcanonical صحيح ذاتي المرجع بكل صفحة، الـhreflang صحيح بدون تداخل لغوي، لا يوجد أخطاء 4xx/5xx، ووضع الصيانة غير مفعّل.

### مشاكل مشتبه بها (تحتاج تأكيد GSC)

قيمة محتوى منخفضة بنظر غوغل لصفحات معاملاتية قصيرة (تسجيل، طلب عرض سعر)، حداثة النطاق وقلة الروابط الخارجية، احتمال تكرار محتوى بين الصفحات القانونية الثلاث.

## حالة OAI-SearchBot و GPTBot

- **OAI-SearchBot:** غير محظور حاليًا (مسموح ضمنيًا). التوصية: إضافة قاعدة صريحة `Allow: /` — لم تُنفَّذ بعد، مجدولة مع قرار GPTBot.
- **GPTBot:** غير محظور حاليًا أيضًا (يستخدم بيانات الموقع لتدريب النماذج، وهذا مختلف عن OAI-SearchBot). **قرار يخصك:** تسمح، تمنع، أو تترك الوضع الحالي الضمني. لم يُغيَّر شي بدون قرارك.

## هل Build مفهومة بوضوح للذكاء الاصطناعي؟

**لا، ليس تمامًا.** الموقع بالكامل (الميتاداتا، الصفحة الرئيسية، الـSchema) يصف Build بشكل ثابت كـ"مورد مواد بناء مباشر"، بينما التموضع الرسمي بالبريف يصفها كـ"شركة توريد وإدارة مشتريات" تعمل مع موردين محليين ودوليين. هذا **تعارض حقيقي بين الموقع والتموضع الرسمي**، مو تعارض بين صفحات الموقع نفسه (الموقع متسق داخليًا، بس متسق على الوصف الخطأ). لم يُغيَّر — يحتاج موافقتك لأنه تغيير محتوى ظاهر لكل الزوار، مو إصلاح تقني.

كذلك: لا توجد صفحة "من نحن" أو "كيف نعمل" إطلاقًا، ولا صفحات مستقلة للفئات الثمانية الرسمية (موجودة فقط ككروت بالصفحة الرئيسية بدون روابط)، ولا يوجد أي ذكر للتوريد الدولي رغم شعار "Sourcing the World".

## الملفات التي تغيّرت

`app/(site)/layout.tsx`، `app/ar/layout.tsx`، `app/maintenance/layout.tsx` (معدّلة) — `app/layout.tsx` (محذوف) — ملفات جديدة: `lib/fonts.ts`، `lib/metadata.ts`، `components/analytics/gtm.tsx`، `components/errors/*`، `app/global-error.tsx`، وملفات `not-found.tsx`/`error.tsx` لكل لغة. التفاصيل الكاملة بـ`seo-fix-log.md`.

## الروابط المتأثرة

كل صفحات الموقع العامة (14 رابط بالـsitemap + صفحات الصيانة/الأخطاء) — التفاصيل بـ`seo-url-inventory.md`.

## نتائج الاختبار قبل/بعد

`next build`: كل الصفحات تحوّلت من `ƒ Dynamic` إلى `○ Static`. `Cache-Control` تحوّل من `private, no-store` إلى `s-maxage=31536000`. `<html lang/dir>` صحيح بكل صفحة. صفحات 404 ترجع 404 حقيقي بدون تكرار وسم `<html>` (اكتُشف وأُصلح خطأ محاولة أولى كانت تنتج وسم `<html>` مزدوج). `lint` و`build` نظيفين.

## نتائج الأداء

لم يتم قياس Core Web Vitals ميدانيًا (يحتاج Lighthouse/CrUX حقيقي غير متاح بهذه البيئة). الإصلاح الرئيسي (إلغاء الـSSR الإجباري) من المتوقع يحسّن TTFB وLCP، لكن هذا غير مقاس مباشرة. تفاصيل الصور والخطوط بـ`seo-performance-report.md`.

## التوصيات التي تحتاج موافقتك (لم تُنفَّذ أي منها)

1. إعادة صياغة الوصف من "مورد" إلى "شركة توريد ومشتريات" — بكل الميتاداتا والصفحة الرئيسية والـSchema.
2. إنشاء صفحة "من نحن/كيف نعمل".
3. إنشاء صفحات مستقلة للفئات الثمانية الرسمية.
4. قرار سياسة GPTBot.
5. إضافة عنوان/موقع حقيقي لـSchema.org `HomeAndConstructionBusiness` أو إزالة هذا النوع.
6. إضافة أحداث تتبع تحويل حقيقية (GA4) — لا يوجد أي منها حاليًا بالكود.
7. تصحيح إعادة التوجيه المؤقتة على نطاق `build.sa` (خارج الريبو، بلوحة Vercel).

التفاصيل الكاملة بـ`approval-required-recommendations.md`.

## معلومات ما زلت أحتاجها منك

- وصول لـSearch Console (Coverage + Crawl Stats + URL Inspection) لتأكيد السبب الحقيقي لضعف الفهرسة.
- قرارك بخصوص GPTBot.
- تأكيد التموضع الرسمي (مورد أم شركة توريد ومشتريات) قبل أي تعديل محتوى ظاهر.
- هل يوجد عنوان مكتب حقيقي تحب تظهره بالـSchema؟
- رابط preview على Vercel لقياس أداء حقيقي (Lighthouse) قبل الدمج.

## أهم 5 إجراءات لتحسين SEO التقليدي

1. الحصول على وصول Search Console وتحديد سبب استبعاد كل رابط من الـ11 غير المفهرس بدقة.
2. تصحيح إعادة توجيه apex domain (307→308/301).
3. إنشاء صفحات الفئات الثمانية (أكبر فجوة كلمات مفتاحية).
4. إنشاء صفحة "من نحن/كيف نعمل".
5. إضافة أحداث تحويل GA4 حقيقية لقياس أثر أي تحسين لاحقًا.

## أهم 5 إجراءات لتحسين ظهور الموقع بمحركات البحث بالذكاء الاصطناعي

1. حسم قرار GPTBot وإضافة قاعدة OAI-SearchBot صريحة بـrobots.txt.
2. توحيد وصف الكيان (مورد ↔ شركة توريد ومشتريات) بكل نقاط الظهور.
3. صفحة "من نحن/كيف نعمل" بمحتوى حقيقي قابل للاقتباس.
4. صفحات الفئات الثمانية (تمنح الذكاء الاصطناعي محتوى محدد للاستشهاد به لكل فئة).
5. ذكر التوريد الدولي فعليًا بالمحتوى (يطابق شعار "Sourcing the World" الحالي غير المدعوم بمحتوى).

## خطة الالتزامات (commits) المقترحة

هذا الفرع يحتوي حاليًا commit واحد فقط (`fix(seo): split root layout per locale...`) لأن التغييرات المنفَّذة فعليًا محدودة بالنقطتين المتفق عليهما. أي إصلاحات إضافية معتمدة لاحقًا (OAI-SearchBot/GPTBot، robots.txt) تُضاف كـcommit منفصل خاص بها، ولا تُدمَج مع تغييرات المحتوى المعتمدة لاحقًا.

## عنوان ووصف Pull Request مقترح (عند الجاهزية للمراجعة، وليس للدمج التلقائي)

**العنوان:** `fix(seo): restore static rendering + RTL html attribute (locale root layout split)`
**الوصف:** يشرح المشكلة المؤكدة (SSR إجباري بلا كاش)، الحل (root layouts منفصلة لكل لغة)، ونتائج الاختبار المحلي (build + curl). يوضح صراحة أن باقي بنود التدقيق (التموضع، الصفحات الجديدة، سياسة GPTBot) موثّقة بملفات منفصلة بانتظار قرارك، ولم تُدمج بهذا الـPR.

## قيود

لا يوجد ضمان لأي نتيجة ترتيب محدد بغوغل، أو ظهور/استشهاد بـChatGPT أو أي نظام ذكاء اصطناعي — هذا خارج قدرة أي تحسين تقني على ضمانه.
